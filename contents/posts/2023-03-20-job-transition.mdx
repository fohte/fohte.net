株式会社シンクロ・フードを退職し、2023-03-20 付けでウォンテッドリー株式会社に入社しました。

# なぜ Wantedly に転職したのか

前職には新卒で入社してから 6 年弱在籍し、やりきった感があった、というのが最大の理由です。  
もちろん、まだやること・やりたかったことは少なくありませんが、成長曲線で考えると緩やかなカーブになっていて成長が止まった感覚を覚えていました。

また、自分の今後のキャリアを考えると、ここ数年先は IC (Individual Contributor) としてキャリアを積みたいという目的もあります。  
より技術的な挑戦ができる環境に身を置き、技術力をもっと上げたいという欲求がありました。

そこで、更なるスキルアップのために転職を検討し始め、Wantedly とご縁があり入社する運びとなりました。

# 転職先に希望していた条件

上述した通り、IC としてのキャリアを積める環境として、以下を希望していました。

- 技術に対して会社として理解があること。重きを置いていること
- エンジニアブログを活発に書くなど、社外への技術発信も精力的に行っていること

また、仕事に対するモチベーションを高めるために、以下もそれなりに強く希望していました。

- 自分がプロダクトに対して興味を持てること

# シンクロ・フードでは何をやっていたのか

新卒入社後は基盤チームに配属され開発基盤の改善などを行い、その後は SRE チームとしてクラウドインフラの設計や構築を主として行っていました。

わかりやすいものでは以下のようなことをやっていました。

- 社内 Rails アプリケーションのリファクタリング
- [Ridgepole](https://github.com/ridgepole/ridgepole) の導入
- 密結合アプリケーションの monorepo 化
  - [依存しあう複数の Git リポジトリを git subtree で monorepo 化する - シンクロ・フード エンジニアブログ](https://tech.synchro-food.co.jp/entry/2018/03/30/115514)
- CI 環境の構築
  - [Drone CI で手軽に Docker コンテナで並列にジョブをぶん回す環境を構築した - シンクロ・フード エンジニアブログ](https://tech.synchro-food.co.jp/entry/2018/09/18/184426)
- React + Flowtype でのアプリケーション開発
  - [Flow で静的型付けしながらフロントエンド開発した話 - シンクロ・フード エンジニアブログ](https://tech.synchro-food.co.jp/entry/2018/03/20/214958)
- EC2 インスタンス上で直接稼働していたアプリケーションをコンテナに移行
- AWS アカウントの分離
  - [AWS Organizations で複数の AWS アカウントを良い感じに運用する - シンクロ・フード エンジニアブログ](https://tech.synchro-food.co.jp/entry/2019/12/09/190000)
- OpenSearch の導入のための設計や検証 (退職前の直近の業務)
  - [OpenSearch の導入による検索システム改善のための認証・認可設計と負荷検証結果 - シンクロ・フード エンジニアブログ](https://tech.synchro-food.co.jp/entry/2023/03/08/183000)

# 最後に

事業に貢献しながらも IC として成長できるようにより一層頑張っていくぞという気持ちです。

また、前職のシンクロ・フードの皆さんには大変お世話になりました。  
社内向けにも書きましたが、最初に選んだ会社がここで良かったと心の底から考えています。  
技術的な挑戦をしても許される環境で本当に好き勝手やっていたのに、受け入れてもらえるだけでなくポジティブな評価もたくさんしていただいたことには特に感謝しています。  
ポジティブな評価だけでなく、自分に足りない部分も指摘してもらい、一人の人間としても成長できたと考えています。  
6 年弱、本当にありがとうございました。
